Type

Validates that a value is of a specific data type. For example, if a variable should be an array, you can use this constraint with the array type option to validate this.

Applies to

property or method

Options

  • groups

  • message

  • payload

  • type

Class

Type [https://github.com/symfony/symfony/blob/master/src/Symfony/Component/Validator/Constraints/Type.php]

Validator

TypeValidator [https://github.com/symfony/symfony/blob/master/src/Symfony/Component/Validator/Constraints/TypeValidator.php]

Basic Usage

This will check if id is an instance of Ramsey\Uuid\UuidInterface, firstName is of type string (using is_string [https://www.php.net/manual/en/function.is-string.php] PHP function), age is an integer (using is_int [https://www.php.net/manual/en/function.is-int.php] PHP function) and accessCode contains either only letters or only digits (using ctype_alpha [https://www.php.net/manual/en/function.ctype-alpha.php] and ctype_digit [https://www.php.net/manual/en/function.ctype-digit.php] PHP functions).

    // src/Entity/Author.php
namespace App\Entity;

use Symfony\Component\Validator\Constraints as Assert;

class Author
{
    /**
     * @Assert\Type("Ramsey\Uuid\UuidInterface")
     */
    protected $id;

    /**
     * @Assert\Type("string")
     */
    protected $firstName;

    /**
     * @Assert\Type(
     *     type="integer",
     *     message="The value {{ value }} is not a valid {{ type }}."
     * )
     */
    protected $age;

    /**
     * @Assert\Type(type={"alpha", "digit"})
     */
    protected $accessCode;
}

    // src/Entity/Author.php
namespace App\Entity;

use Ramsey\Uuid\UuidInterface;
use Symfony\Component\Validator\Constraints as Assert;
use Symfony\Component\Validator\Mapping\ClassMetadata;

class Author
{
    public static function loadValidatorMetadata(ClassMetadata $metadata)
    {
        $metadata->addPropertyConstraint('id', new Assert\Type(UuidInterface::class));

        $metadata->addPropertyConstraint('firstName', new Assert\Type('string'));

        $metadata->addPropertyConstraint('age', new Assert\Type([
            'type' => 'integer',
            'message' => 'The value {{ value }} is not a valid {{ type }}.',
        ]));

        $metadata->addPropertyConstraint('accessCode', new Assert\Type([
            'type' => ['alpha', 'digit'],
        ]));
    }
}

Options

groups

type: array | string

It defines the validation group or groups this constraint belongs to. Read more about validation groups.

message

type: string default: This value should be of type {{ type }}.

The message if the underlying data is not of the given type.

You can use the following parameters in this message:

Parameter

Description

{{ type }}

The expected type

{{ value }}

The current (invalid) value

{{ label }}

Corresponding form field label

New in version 5.2: The {{ label }} parameter was introduced in Symfony 5.2.

payload

type: mixed default: null

This option can be used to attach arbitrary domain-specific data to a constraint. The configured payload is not used by the Validator component, but its processing is completely up to you.

For example, you may want to use several error levels to present failed constraints differently in the front-end depending on the severity of the error.

type

type: string or array [default option]

This required option defines the type or collection of types allowed for the given value. Each type is either the FQCN (fully qualified class name) of some PHP class/interface or a valid PHP datatype (checked by PHP’s is_() functions):

  • array [https://www.php.net/manual/en/function.is-array.php]

  • bool [https://www.php.net/manual/en/function.is-bool.php]

  • callable [https://www.php.net/manual/en/function.is-callable.php]

  • float [https://www.php.net/manual/en/function.is-float.php]

  • double [https://www.php.net/manual/en/function.is-double.php]

  • int [https://www.php.net/manual/en/function.is-int.php]

  • integer [https://www.php.net/manual/en/function.is-integer.php]

  • iterable [https://www.php.net/manual/en/function.is-iterable.php]

  • long [https://www.php.net/manual/en/function.is-long.php]

  • null [https://www.php.net/manual/en/function.is-null.php]

  • numeric [https://www.php.net/manual/en/function.is-numeric.php]

  • object [https://www.php.net/manual/en/function.is-object.php]

  • real [https://www.php.net/manual/en/function.is-real.php]

  • resource [https://www.php.net/manual/en/function.is-resource.php]

  • scalar [https://www.php.net/manual/en/function.is-scalar.php]

  • string [https://www.php.net/manual/en/function.is-string.php]

Also, you can use ctype_*() functions from corresponding built-in PHP extension [https://www.php.net/book.ctype]. Consider a list of ctype functions [https://www.php.net/ref.ctype]:

  • alnum [https://www.php.net/manual/en/function.ctype-alnum.php]

  • alpha [https://www.php.net/manual/en/function.ctype-alpha.php]

  • cntrl [https://www.php.net/manual/en/function.ctype-cntrl.php]

  • digit [https://www.php.net/manual/en/function.ctype-digit.php]

  • graph [https://www.php.net/manual/en/function.ctype-graph.php]

  • lower [https://www.php.net/manual/en/function.ctype-lower.php]

  • print [https://www.php.net/manual/en/function.ctype-print.php]

  • punct [https://www.php.net/manual/en/function.ctype-punct.php]

  • space [https://www.php.net/manual/en/function.ctype-space.php]

  • upper [https://www.php.net/manual/en/function.ctype-upper.php]

  • xdigit [https://www.php.net/manual/en/function.ctype-xdigit.php]

Make sure that the proper locale [https://www.php.net/manual/en/function.setlocale.php] is set before using one of these.
